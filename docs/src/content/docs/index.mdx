<div class="section-heading">
  <h2>Built to Bring Gamers Together</h2>
  <p>Raid Ledger exists to solve one problem: getting your community to actually show up and play together. Less coordination overhead, more game time — all where your community already lives.</p>
</div>

<div class="features-grid">
  <div class="feature-card">
    <h3>Event Scheduling &amp; Recurring Raids</h3>
    <p>Spin up one-off or repeating events — weekly, biweekly, or monthly — with reusable templates and a shared calendar, so raid night books itself instead of living in a pinned message.</p>
  </div>

  <div class="feature-card">
    <h3>Discord-Native Signups &amp; Rosters</h3>
    <p>Members sign up right inside Discord with rich embeds and slash commands. Auto-allocation, bench slots, and tentative-to-confirmed promotion keep every roster filled without manual juggling.</p>
  </div>

  <div class="feature-card">
    <h3>Automated Voice Attendance</h3>
    <p>Attendance is tracked from who actually shows up in voice — no roll-call, no spreadsheet — with two-phase no-show detection that nudges absent players and flags them to the host.</p>
  </div>

  <div class="feature-card">
    <h3>Ephemeral Voice Channels</h3>
    <p>A dedicated voice channel is created automatically for each event and cleaned up when it ends, so your server stays tidy and every raid gets its own room exactly when it's needed.</p>
  </div>

  <div class="feature-card">
    <h3>Running-Late &amp; Host-Delay Controls</h3>
    <p>Attendees can flag that they're running late, and hosts can bump the start by 15 or 30 minutes in one tap — keeping everyone in sync without cancelling confirmations or spamming the channel.</p>
  </div>

  <div class="feature-card">
    <h3>Smart Reminders &amp; Reschedule Flows</h3>
    <p>Batched, de-duplicated DM reminders and one-tap confirm/decline reschedule prompts mean fewer missed raids and far less chasing people down.</p>
  </div>

  <div class="feature-card">
    <h3>Community Lineups &amp; AI Suggestions</h3>
    <p>Let the group nominate and vote on what to play. Common Ground scoring surfaces the games everyone actually wants, and AI-assisted picks (including wildcards) settle "what are we playing tonight?"</p>
  </div>

  <div class="feature-card">
    <h3>Scheduling Polls &amp; Availability Heatmaps</h3>
    <p>Find the time that works for the most people with group availability heatmaps and scheduling polls — complete with deadlines and tiebreakers — so you lock in a slot instead of guessing.</p>
  </div>

  <div class="feature-card">
    <h3>Game Library with Live Deal Pricing</h3>
    <p>A rich game library powered by IGDB pairs every title with live IsThereAnyDeal pricing, plus Steam wishlist and playtime sync, so your community discovers what's trending, owned, and on sale.</p>
  </div>

  <div class="feature-card">
    <h3>Taste Profiles &amp; Characters</h3>
    <p>Every member gets a taste profile and a fun archetype built from real play signals, while players manage characters — including World of Warcraft Classic toons imported via the Blizzard API.</p>
  </div>

  <div class="feature-card">
    <h3>Community Insights &amp; Analytics</h3>
    <p>Admin dashboards reveal attendance trends, event metrics, churn risk, and social cliques — the signals you need to keep your community healthy and growing.</p>
  </div>

  <div class="feature-card">
    <h3>Discord OAuth Sign-In</h3>

<script>
{`
  document.addEventListener('DOMContentLoaded', () => {
    const carousel = document.querySelector('.carousel');
    const track = document.querySelector('.carousel-track');
    const slides = document.querySelectorAll('.carousel-slide');
    const prevBtn = document.querySelector('.carousel-prev');
    const nextBtn = document.querySelector('.carousel-next');
    const dotsContainer = document.getElementById('carousel-dots');
    let current = 0;

    function sizeSlides() {
      const w = carousel.offsetWidth;
      slides.forEach(s => { s.style.width = w + 'px'; s.style.minWidth = w + 'px'; });
      track.style.transform = 'translateX(-' + (current * w) + 'px)';
    }

    function positionButtons() {
      const img = slides[current] && slides[current].querySelector('img');
      if (img) {
        const h = img.offsetHeight;
        prevBtn.style.top = (h / 2) + 'px';
        nextBtn.style.top = (h / 2) + 'px';
      }
    }

    sizeSlides();
    window.addEventListener('resize', () => { sizeSlides(); positionButtons(); });

    slides.forEach((_, i) => {
      const dot = document.createElement('button');
      dot.className = 'carousel-dot' + (i === 0 ? ' active' : '');
      dot.setAttribute('aria-label', 'Go to slide ' + (i + 1));
      dot.addEventListener('click', () => goTo(i));
      dotsContainer.appendChild(dot);
    });

    function goTo(index) {
      current = (index + slides.length) % slides.length;
      const w = carousel.offsetWidth;
      track.style.transform = 'translateX(-' + (current * w) + 'px)';
      document.querySelectorAll('.carousel-dot').forEach((d, i) =>
        d.classList.toggle('active', i === current)
      );
      positionButtons();
    }

    prevBtn.addEventListener('click', () => goTo(current - 1));
    nextBtn.addEventListener('click', () => goTo(current + 1));

    const firstImg = slides[0] && slides[0].querySelector('img');
    if (firstImg && firstImg.complete) { positionButtons(); }
    else if (firstImg) { firstImg.addEventListener('load', positionButtons); }

    let startX = 0;
    track.addEventListener('touchstart', (e) => { startX = e.touches[0].clientX; });
    track.addEventListener('touchend', (e) => {
      const diff = startX - e.changedTouches[0].clientX;
      if (Math.abs(diff) > 50) goTo(current + (diff > 0 ? 1 : -1));
    });
  });
`}
</script>

<div class="section-heading">
  <h2>Quick Start</h2>
  <p>One command runs the whole stack — app, PostgreSQL, and Redis. Mount a volume so your data and nightly backups persist.</p>
</div>

```bash
docker run -d -p 80:80 -v raid-ledger-data:/data ghcr.io/sjdodge123/raid-ledger:main
```

Then open [http://localhost](http://localhost) and check the container logs for your admin credentials. Want to look around first? Install demo data from the Admin Panel to explore a fully-populated community.
