  <h2>Everything Your Community Needs</h2>
  <p>From event planning to attendance tracking, Raid Ledger keeps your gaming community organized.</p>
</div>

<div class="features-grid">
  <div class="feature-card">
    <h3>Events and Scheduling</h3>
    <p>Create one-time or recurring events with natural language time parsing, calendar view, and automatic Discord announcements.</p>
  </div>

  <div class="feature-card">
    <h3>Roster Management</h3>
    <p>MMO-style role assignments (Tank/Healer/DPS), drag-and-drop slot management, bench tracking, and PUG invite system.</p>
  </div>

  <div class="feature-card">
    <h3>Discord Bot Integration</h3>
    <p>Full-featured Discord bot with slash commands for event creation, roster viewing, channel bindings, and voice activity monitoring.</p>
  </div>

  <div class="feature-card">
    <h3>Game Library (IGDB)</h3>
    <p>Browse and filter from a catalog of 200,000+ games powered by IGDB. Genre filtering, cover art, and game-specific event grouping.</p>
  </div>

  <div class="feature-card">
    <h3>Analytics and Metrics</h3>
    <p>Track attendance rates, no-show patterns, game popularity, and community engagement over time.</p>
  </div>

  <div class="feature-card">
    <h3>Smart Notifications</h3>
    <p>Automated event reminders, no-show detection with DM alerts, and configurable notification preferences per user.</p>
  </div>

  <div class="feature-card">
    <h3>Ad-Hoc Voice Events</h3>
    <p>Auto-detect games in voice channels via Discord Rich Presence and create spontaneous events with live activity badges.</p>
  </div>

  <div class="feature-card">
    <h3>Plugin System</h3>
    <p>Extensible plugin architecture — Discord OAuth authentication, Blizzard API integration for WoW character sync, and more to come.</p>
  </div>
</div>

<div class="section-heading">
  <h2>Quick Start</h2>
  <p>Get Raid Ledger running in under a minute.</p>
</div>

```bash
docker run -d -p 80:80 ghcr.io/sjdodge123/raid-ledger:main
```

Then open [http://localhost](http://localhost) and check the container logs for your admin credentials.
