# Tech Debt Backlog

Working document for reviewer findings that don't warrant immediate fixes. Appended by `/build`, `/dispatch`, and `/fix-batch` reviewer reports. Operator triages: file Linear stories for items worth doing, delete the rest.

**Why this exists:** findings used to be auto-filed as Linear `tech-debt:` stories, which created a self-perpetuating queue (reviewer flags items ŌåÆ Lead files story ŌåÆ next batch picks up the story ŌåÆ reviewer flags more items ŌåÆ loop). Findings now land here instead. Nothing in this file should be acted on by an agent without an explicit operator instruction.

## Format for new entries

Append to the bottom under a new dated section. The Lead does this as part of the batch's commits ŌĆö no separate PR.

```
### 2026-05-03 ŌĆö fix/batch-2026-05-02 (PR #XXX)

- **[low]** `path/to/file.ts:42` ŌĆö short description.
  Suggested: one-line fix idea (optional).
- **[med]** `path/other.tsx` ŌĆö description.
```

Severities: `crit` should never land here (those are auto-fixed during review or sent back to dev). Use `high` / `med` / `low` / `nit`.

## Operator workflow

1. Read new entries when reviewing the PR diff.
2. For each entry: file a Linear story manually, OR delete the entry, OR leave it for later.
3. Pruning is part of triage ŌĆö old or duplicate items should be removed, not left to bloat.

## Format for skills that parse this file

A future triage skill (or operator-invoked agent) can rely on:

- Each batch is a level-3 heading: `### YYYY-MM-DD ŌĆö <branch> (PR #<num>)`. Date is ISO. PR ref optional if the PR doesn't exist yet.
- Each finding is a single bullet starting with `- **[<sev>]**` where `<sev>` Ōłł {`high`, `med`, `low`, `nit`}.
- File path follows in single backticks; `path:line` is preferred over `path` alone.
- Description is the rest of the bullet up to optional `Suggested:` line indented two spaces.
- The append marker (`<!-- agents append below this line -->`) divides header from entries ŌĆö skills should read from there forward.

A skill's job is typically: parse ŌåÆ group duplicates by file path ŌåÆ propose Linear stories grouped by area label ŌåÆ present for operator approval. Never auto-file.
